# Other Project Bootstrap

Use this repo as the website-design brain for a different project directory when you want 21st-informed direction that is based on actual discovered templates/components, not just cached category names.

## Canonical Prompt

```text
Use Website Design Intelligence from:
C:\Users\abe\OneDrive\website-design-intelligence

Target project:
<replace with project path>

Treat 21st as the primary design source for this website.
Do not assume the local 21st cache is full coverage just because it exists.

Before planning or coding:
1. Check private 21st cache coverage.
2. Determine the relevant 21st categories for this brief.
3. If coverage is incomplete for those categories, inspect live 21st category pages and item inventories.
4. Enumerate the actual 21st items found.
5. Separate:
   - cached items
   - live-discovered items
   - selected references
6. Use private prompt/code recipes when available to understand how the chosen patterns are built.

Then:
7. Analyze the patterns you found.
8. Propose the strongest direction.
9. Adapt that direction to the target brand and website goals.
10. Give a build spec before coding.

Do not claim broad 21st access unless you can name the actual templates/components you inspected.
```

## Expected Behavior

- The agent should use cache as a starting point, not as proof of full 21st visibility.
- If the project depends on 21st direction, the agent should discover actual items for the relevant categories before making strong recommendations.
- The output should distinguish cache truth from live discovery truth.
- When prompt/code recipes are available for collected 21st items, the agent should use them to understand, adapt, and improve the pattern instead of cloning it blindly.
- The final website direction should adapt 21st patterns into the target brand instead of cloning them.
