# Workflow

## Core pipeline

1. Ingest a business brief.
2. Filter reference metadata by section type, industry, site type, tone, and strengths.
3. Select the strongest reference for each required section.
4. Compose a unified visual direction from the chosen sections.
5. Apply brand tokens and copy direction rules.
6. Output a compact build spec for a downstream coding agent.

## Why this is spec first

Spec-first output keeps token use lower and makes the system easier to review before generating code.

## Compliance rule

Use reference patterns as inspiration only. Do not store or reproduce third-party code, copy, logos, or assets.
